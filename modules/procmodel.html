Module procmodel

Procedural models for basic shapes

Module that generates simple procedural models of simple shapes
Currently only spheres, cylinders, planes & cubemarched models are implemented

LK3D.DeclareProcSphere(name, coli, rowi, szx, szy, uvm, invert)
Declares a procedural sphere model

Parameters:

  • name string LK3D model name
  • coli number Longitude steps
  • rowi number Latitude steps
  • szx optional number X Size
  • szy optional number Y Size
  • uvm optional number UV map scale
  • invert optional bool whether we should invert the faces or not

Usage:

    LK3D.DeclareProcSphere("simple_ball", 8, 6, 1, 1, 1, false)

LK3D.DeclareProcPlane(name, psx, psy, itrx, itry, distortfunc, uvScale)
Declares a procedural plane model

Parameters:

  • name string LK3D model name
  • psx number X Size
  • psy number Y Size
  • itrx number X Vertex count
  • itry number Y Vertex count
  • distortfunc optional function Distort function for altering positions, refer to usage
  • uvScale optional number How much to scale the UVs of this plane

Usage:

    LK3D.DeclareProcPlane("cool_terrain", 8, 8, 16, 16, function(x, y)
 local spx = LK3D.Simplex2D(x, y, 5623)
 return Vector(x, spx, y)
end)
LK3D.DeclareCubeMarchModel(name, detail, sz, func)
Declares a cubemarched model

Parameters:

  • name string LK3D model name
  • detail number Detail level of the model
  • sz vector Scale of the model
  • func function SDF function for the model

Usage:

    -- if you don't know what an SDF is, this is a good read, https://jamie-wong.com/2016/07/15/ray-marching-signed-distance-functions/
-- in this case, i used the sphere SDF from https://iquilezles.org/articles/distfunctions/
-- also UVs aren't implemented properly on this
LK3D.DeclareCubeMarchModel("sphere_marched", 4, Vector(1, 1, 1), function(pos)
	 return pos:Length() - .7
end)
